Salesforce Integration Guide

A comprehensive reference for building typed JSON wrapper classes and parsing callouts in Salesforce Apex.

Introduction to Salesforce API Integrations

When building integrations within the Salesforce platform, Apex developers frequently interact with external REST APIs. These external APIs communicate by exchanging JSON payloads. To consume these responses inside Salesforce, you must parse the JSON format back into native Apex data structures.

While Apex supports untyped JSON deserialization (using generic maps of strings to objects), using statically typed Apex Wrapper Classes is the industry-recognized best practice. This approach provides:

  • Compile-time safety: Errors are caught during development or deployment, rather than crashing at runtime.
  • IDE autocomplete: Properties are explicitly typed, making code easier to write and maintain in tools like VS Code.
  • Self-documenting APIs: The wrapper structure models the external JSON schema, making integrations straightforward to understand for team members.

Note: By automating the creation of wrapper and unit test classes, json2apex helps developers eliminate hours of repetitive boilerplate coding.

Statically Typed Deserialization in Action

Salesforce provides the native JSON.deserialize() method to parse JSON content into a specific Apex class. Below is the general pattern used to call and deserialize an API payload: 

// 1. Fetch the JSON response from Http Callout
HttpResponse response = http.send(request);
String jsonBody = response.getBody();

// 2. Deserialize the response into the Apex wrapper class
MyWrapperClass data = (MyWrapperClass) JSON.deserialize(jsonBody, MyWrapperClass.class);

// 3. Directly access strongly typed fields
System.debug('User ID: ' + data.userId);
System.debug('User Email: ' + data.contactInfo.email);

Handling Salesforce Reserved Keywords

One of the primary challenges when converting external JSON payloads to Apex is **Reserved Keywords**. External services (written in Node.js, Python, or Go) frequently use property names that are protected, reserved compiler words in Apex.

Examples of reserved keywords in Apex include: class, public, private, limit, group, currency, object, sharing, and void.

If you try to declare an Apex class variable with these names: 

// THIS WILL CAUSE A COMPILER ERROR IN SALESFORCE
public class ProductInfo {
    public String name;
    public String class; // Fatal compiler error: Reserved keyword
}

How json2apex Automatically Sanitizes Keywords

To prevent these compiler failures, json2apex automatically detects reserved terms in your JSON properties and prefixes them with an underscore (e.g. class is written as _class).

To successfully deserialize this sanitized property, you can replace the key in the raw JSON string immediately before parsing: 

// Pre-deserialize replacement pattern for reserved keywords
String sanitizedJson = jsonString.replaceAll('"class":', '"_class":');
MyWrapperClass result = (MyWrapperClass) JSON.deserialize(sanitizedJson, MyWrapperClass.class);

Our tool automatically generates this replacement helper logic directly inside your Apex Wrapper class methods if reserved keywords are detected, saving you from writing custom regex maps.

Avoiding Common JSON_PARSER_ERRORs

Runtime parsing exceptions are common during Salesforce integrations. Understanding their cause is the best way to prevent them:

1. "JSON_PARSER_ERROR: Expected Array, Got Object"

This error happens when your Apex wrapper declares a property as a List (e.g. List<Product>), but the external API dynamically returns a single JSON object instead of an array.

Fix: Ensure the external service is configured to always return an array, or parse the property dynamically using JSON.deserializeUntyped() to examine its structure at runtime before mapping it.

2. Type Mismatches

If an API payload changes a field dynamically—for example, returning a String ("12.50") in one request and a Decimal (12.50) in another—Apex's strict typing system will throw a runtime error.

Fix: Declare variables as standard String or Decimal structures depending on the base API schema. If fields are volatile, parsing them into generic Object formats offers maximum stability.

Full Integration Walkthrough & Code Example

Let's review a complete, real-world example mapping an external user detail response.

Step 1: The Input JSON Payload

{
  "id": 1045,
  "name": "Jane Developer",
  "email": "jane@example.com",
  "class": "Premium Tier",
  "roles": ["Administrator", "Developer"],
  "meta": {
    "last_login": "2026-07-17T12:00:00Z",
    "active": true
  }
}

Step 2: Generated Apex Wrapper Class

Notice how the reserved keyword class is automatically sanitized to _class, and a custom parse() utility method is included to process the replacement rule: 

public class UserResponse {

    public Integer id;
    public String name;
    public String email;
    public String _class; // Sanitized from "class"
    public List<String> roles;
    public MetaInfo meta;

    public class MetaInfo {
        public String last_login;
        public Boolean active;
    }

    // Custom parse method that cleans reserved keywords
    public static UserResponse parse(String jsonStr) {
        if (jsonStr == null) return null;
        // Clean keyword
        String cleanedJson = jsonStr.replaceAll('"class":', '"_class":');
        return (UserResponse) JSON.deserialize(cleanedJson, UserResponse.class);
    }
}

Step 3: Generated Unit Test Class

Writing tests in Salesforce is mandatory to move code to production (75% minimum coverage threshold). The test class verifies proper parsing and asserts details: 

@isTest
private class UserResponse_Test {

    @isTest
    static void testParse() {
        String jsonStr = '{"id": 1045, "name": "Jane Developer", "email": "jane@example.com", "class": "Premium Tier", "roles": ["Administrator"], "meta": {"last_login": "2026-07-17T12:00:00Z", "active": true}}';
        
        Test.startTest();
        UserResponse obj = UserResponse.parse(jsonStr);
        Test.stopTest();

        System.assert(obj != null);
        System.assertEquals(1045, obj.id);
        System.assertEquals('Jane Developer', obj.name);
        System.assertEquals('Premium Tier', obj._class);
        System.assertEquals(true, obj.meta.active);
    }
    
    @isTest
    static void testParseNull() {
        UserResponse obj = UserResponse.parse(null);
        System.assertEquals(null, obj);
    }
}
